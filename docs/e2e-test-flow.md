# E2E Test Flow

Пошаговый сценарий ручного тестирования полного флоу от открытия бота до завершения заказа.

## Подготовка

- [ ] Бот запущен в Railway (Online)
- [ ] API запущен в Railway (Online)
- [ ] В БД есть повара с verificationStatus='approved'
- [ ] VITE_BOT_USERNAME задан в Vercel env vars
- [ ] Telegram Payments в тестовом режиме

## Флоу 1 — Заказчик создаёт заказ

- [ ] /start → видим приветствие с кнопкой "Найти повара"
- [ ] Открываем mini app → каталог показывает поваров
- [ ] Тапаем на карточку повара → открывается профиль
- [ ] Нажимаем "Заказать" → форма заказа
- [ ] Заполняем: тип, дата, количество людей → создаём заказ
- [ ] Заказ создан со статусом `awaiting_payment`
- [ ] Повар получает уведомление в бот

## Флоу 2 — Повар устанавливает цену

- [ ] Входим как повар → видим уведомление о новом заказе
- [ ] Открываем заказ → видим форму ввода цены
- [ ] Вводим цену → нажимаем "Подтвердить цену"
- [ ] Заказчику приходит уведомление что цена установлена

## Флоу 3 — Оплата

- [ ] Заказчик открывает заказ → кнопка "Оплатить" активна
- [ ] Нажимаем "Оплатить" → открывается Telegram invoice
- [ ] Оплачиваем тестовой картой `4242 4242 4242 4242`
- [ ] Статус меняется на `paid`
- [ ] Повар получает уведомление об оплате

## Флоу 4 — Завершение

- [ ] Меняем `scheduled_at` заказа в прошлое (Railway Query):
  ```sql
  UPDATE orders SET scheduled_at = NOW() - INTERVAL '1 day' WHERE id = <order_id>;
  ```
- [ ] Заказчик видит кнопки "Всё прошло хорошо" / "Есть проблема"
- [ ] Нажимаем "Всё прошло хорошо" → статус `completed`
- [ ] Форма отзыва → ставим оценку → отправляем
- [ ] Рейтинг повара обновился

## Флоу 5 — Чат повар-заказчик

- [ ] Повар нажимает "Написать заказчику" в заказе
- [ ] Открывается бот с активным чатом
- [ ] Пишем сообщение → заказчик получает его в боте
- [ ] Заказчик отвечает → повар получает ответ
