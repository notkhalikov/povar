# E2E Manual Testing Checklist

## Подготовка

```bash
# 1. Запустить API и заполнить БД тестовыми данными
cd apps/api
npx tsx src/db/seed.ts

# 2. Получить JWT для тестовых аккаунтов (NODE_ENV=development)
curl http://localhost:3000/dev/token/2000001   # Тестовый Заказчик 1
curl http://localhost:3000/dev/token/2000002   # Тестовый Заказчик 2
curl http://localhost:3000/dev/token/1000001   # Нино Беридзе (повар)
curl http://localhost:3000/dev/token/1000002   # Тамара Кварацхелия (повар)

# 3. Сбросить БД и пересидировать заново
curl -X POST http://localhost:3000/dev/reset
```

## Чеклист регрессии после изменений

Перед мержем любого PR проверить:

- [ ] Аутентификация через initData не сломана (тестовый аккаунт логинится)
- [ ] GET /chefs возвращает только активных, верифицированных, не забаненных поваров
- [ ] Создание заказа → статус awaiting_payment
- [ ] POST /reviews обновляет ratingCache повара
- [ ] Спор закрывает заказ в dispute_pending
- [ ] /dev/token работает только при NODE_ENV=development
- [ ] POST /dev/reset очищает и пересидирует без ошибок
